# 买啥 MaiSha — Design Document

**Date:** 2026-04-22
**Status:** Draft (awaiting user review)
**Scope:** v1 only. v2/v3 roadmap included for context but not in scope.

---

## 1. 背景与目标

### 1.1 问题

家庭日常采购痛点：
- 平时在家想到要买的东西（菜、日用品），到超市就忘了或漏掉
- 夫妻双方各自记在脑子里，没对齐，容易重复买或漏买
- 采购路线是"跑几家店"（比如菜场 + 盒马 + 山姆），需要按店组织清单
- 已有大而全的 app（AnyList、Paprika）功能过多、学习成本高

### 1.2 目标

做一个**轻量、共享、按超市组织**的购物清单小程序：
- 夫妻两人共用一张清单，实时同步
- 按"去哪家店买"分组，方便到店对照
- 零注册、零登录、微信直接用
- 打开 → 添加 / 打勾 → 分享给对方，三步核心流程无任何负担

### 1.3 非目标（v1 不做）

- 菜谱 / 下周菜单规划（放到 v3）
- 库存 / 拍照生产日期 / 过期提醒（放到 v2）
- 多清单（只支持单一共享清单）
- 账号 / 登录 / 密码
- 支付 / 订单 / 电商跳转
- 离线使用（MVP 依赖云同步；短时断网靠本地 state 兜底）

### 1.4 参考对标

- **Bring!**：视觉分类、轻量交互
- **AnyList**：分享共享列表
- **Paprika**：我们特意不做菜谱（给 v3）
- **Nowaste**：我们特意不做过期追踪（给 v2）

## 2. 用户与场景

### 2.1 用户画像

- **主要用户（发起者）**：在家记想买的东西、规划采购的一方
- **协同用户（被邀请者）**：主要负责执行采购、打勾确认买了的一方
- 两人都会：添加、勾选、删除物品

### 2.2 核心场景

| # | 场景 | 触发 | 核心操作 |
|---|---|---|---|
| 1 | 在家想起要买的东西 | 日常 | 打开 → 添加物品（输入/选常买）|
| 2 | 出门前检视清单 | 采购前 | 浏览、调整超市归属 |
| 3 | 超市现场对照购买 | 采购中 | 切到对应超市 → 逐项打勾 |
| 4 | 让对方帮买 | 采购前 | 📤 邀请（首次）或分享卡片 |
| 5 | 同步知道对方买了什么 | 采购中 | 打开 app 看实时状态 |
| 6 | 清空已购，开始下一轮 | 采购后 | "清空已购" |

## 3. v1 功能范围

### 3.1 核心功能

1. **单一共享清单**：两个微信用户共享同一张清单
2. **按超市分组**：主视图按超市卡片分组（盒马 / 菜场 / 山姆 / 便利店 / 未分类）
3. **按品类副分组**：每个超市内部按品类（蔬菜 / 乳制品 / 日用等）配 emoji 做视觉锚点
4. **添加物品**：
   - 输入框带自动补全 + 拼音搜索
   - "常买" chips（基于使用频率的 top 6 物品）
   - "最近添加" chips（最近 5 次添加的完整条目）
   - 必填：名称；可选：备注（品牌/规格）、数量、超市、品类
5. **勾选/取消勾选**：点击 item → 打勾（变灰 + 划掉）；再点击 → 取消
6. **编辑/删除**：长按 item → 弹出菜单（编辑、删除、复制到下次）
7. **清空已购**：顶部 ⋯ 菜单 → 一键删除所有已勾选项
8. **邀请老公**：顶部 📤 → 生成微信分享卡片 → 发给他 → 他点开自动加入
9. **实时同步**：两端任意操作，对方 1 秒内看到变化
10. **超市管理**：设置页可以增/删/改超市项（默认提供 6 个，用户可自定义）

### 3.2 数据模型

**云数据库（wxcloud）两个 collection：**

#### `lists`
```js
{
  _id: "auto-id",                    // listId
  name: "家里",                       // 用户可改
  ownerOpenid: "ouid_xxx",
  memberOpenids: ["ouid_xxx", ...],  // 含 owner
  supermarkets: [                    // 超市配置（可编辑）
    { id: "hm",   name: "盒马",      emoji: "🛒" },
    { id: "cc",   name: "菜场",      emoji: "🥬" },
    { id: "sm",   name: "山姆",      emoji: "🏬" },
    { id: "jd",   name: "京东到家",  emoji: "📦" },
    { id: "cvs",  name: "便利店",    emoji: "🏪" },
    { id: "none", name: "未分类",    emoji: "❓" }
  ],
  createdAt, updatedAt
}
```

#### `items`
```js
{
  _id: "auto-id",
  listId: "lists._id",
  name: "牛奶",                      // 核心名（用于归类/emoji 匹配）
  note: "伊利 1L",                   // 可选：品牌/规格
  quantity: "2瓶",                   // 可选：数量+单位
  supermarket: "hm",                 // 引用 lists.supermarkets[].id；默认 "none"
  category: "乳制品",                // 自动推断 or 手动
  categoryEmoji: "🥛",
  checked: false,
  checkedAt: null,
  createdBy: "ouid_xxx",             // 仅内部记录，不在 UI 上显示
  createdAt, updatedAt
}
```

**"常买"chips 的数据来源（客户端本地存储）：**

v1 不把频率统计放在云库里。每次用户添加物品时，小程序调用 `wx.setStorageSync` 在**本地**累计：

```js
// 本地 key: `frequent_${openid}`，value: 数组
[
  { name, note, supermarket, categoryEmoji, count, lastUsedAt },
  ...
]
```

- 添加 chips 时按 `count desc, lastUsedAt desc` 取 top 6
- 添加物品 → 聚合键（`name+note+supermarket`）已存在则 `count++`；不存在则插入
- 重装小程序或换设备会丢失频率记录（可接受：用户用几天后会自然重建），核心清单数据依然在云端安全

> 选择本地而非云端的理由：(1) 个性化数据，不必共享；(2) 无需额外集合和读写开销；(3) v1 最快速实现路径。

### 3.3 默认品类-Emoji 映射

内置关键词 → emoji 映射表（硬编码，v1 不做 UI 编辑）：

| 品类 | Emoji | 关键词示例 |
|---|---|---|
| 蔬菜 | 🥬 | 青菜、白菜、西红柿、黄瓜、土豆、胡萝卜… |
| 水果 | 🍎 | 苹果、香蕉、橙子、葡萄、草莓… |
| 肉蛋 | 🥩 | 猪肉、牛肉、鸡肉、鸡蛋… |
| 乳制品 | 🥛 | 牛奶、酸奶、奶酪、黄油… |
| 主食 | 🍚 | 米、面条、饺子、馒头… |
| 烘焙 | 🍞 | 面包、吐司、蛋糕… |
| 调料 | 🧂 | 盐、糖、酱油、醋、料酒… |
| 零食 | 🍪 | 饼干、巧克力、薯片… |
| 饮料 | 🥤 | 可乐、茶、咖啡、果汁… |
| 日用 | 🧻 | 纸巾、洗衣液、牙膏、洗发水… |
| 其他 | 📦 | （未匹配的默认归到这里）|

匹配逻辑：对物品 `name` 做子串包含检查，命中任一关键词 → 归类。匹配不到 → "其他"。用户也可以在编辑物品时手动改。

### 3.4 冲突处理规则

| 情况 | 规则 |
|---|---|
| 两人同时勾选同一项 | Last-write-wins（`checkedAt` 最新的为准，云端自动用 `updatedAt` 排序）|
| 两人同时添加同名物品 | 两条都保留（显示重复项，避免误合并）|
| 一方删除、另一方编辑 | 删除优先（`items` 直接从云库消失）|
| 网络断开期间的本地改动 | 缓存在本地 state，恢复后重试写入（wxcloud 内置重试）|

## 5. 技术栈与架构

### 5.1 技术栈

| 层 | 选型 | 理由 |
|---|---|---|
| 前端 | 微信小程序原生（WXML / WXSS / **TypeScript**） | 最佳 wxcloud 集成；无需抽象层；热更新最快；TS 提供类型安全（对 lists/items 数据模型尤其有价值） |
| 组件库 | Vant Weapp | 覆盖 90% UI 需求，视觉一致 |
| 后端 | wxcloud（微信云开发） | 免运维、免费额度充足、天然和小程序打通 |
| 数据库 | wxcloud 云数据库（MongoDB-like） | 文档型，灵活，原生支持 watch 实时订阅 |
| 云函数 | wxcloud 云函数（Node.js） | 用于 `joinList`、`clearChecked` 等原子操作 |
| 身份 | 微信 openid | 每个微信用户自带唯一 ID，零登录 |
| 开发工具 | 微信开发者工具（Windows） | 官方 IDE，免费，含模拟器、调试、发布工具 |

### 5.2 小程序页面结构

```
pages/
├── index/          主清单页面
├── add/            添加物品（也可用 popup 实现，决定在实现时定）
├── edit-item/      编辑单个物品
├── manage-markets/ 管理超市
├── settings/       设置（含"退出清单""邀请老公"入口）
└── onboarding/     首次打开引导（欢迎 + 说明 + 自动建清单）

cloudfunctions/
├── joinList/       接收 listId + 调用者 openid，把 openid 加入 memberOpenids
├── clearChecked/   原子删除所有 checked=true 的物品（确保两人同时操作不冲突）
└── getInitData/    首次打开时返回清单信息（并处理首次建清单逻辑）

components/
├── item-row/       单个物品行
├── supermarket-card/ 超市卡片
├── add-sheet/      添加面板
└── chip/           通用 chip
```

### 5.3 实时同步实现

主清单页打开时：

```js
const db = wx.cloud.database();
const watcher = db.collection('items')
  .where({ listId: this.data.listId })
  .watch({
    onChange: (snapshot) => {
      this.setData({ items: snapshot.docs });
    },
    onError: (err) => {
      wx.showToast({ title: '同步断开，重试中…' });
      // 自动重连由 SDK 处理
    }
  });
```

页面销毁时 `watcher.close()`。

### 5.4 发布路径

1. 注册**个人类型**小程序账号（[mp.weixin.qq.com](https://mp.weixin.qq.com)，免备案，秒开通）
2. 开通云开发（同一后台，免费）
3. 完成 v1 实现
4. 提交审核（家用工具类通常 1-3 天审核通过）
5. 发布后两人用自己的微信搜"买啥"或扫二维码进入

### 5.5 限制与注意

- 个人小程序：
  - 不能使用微信支付（我们不需要）
  - 不能涉及广告、电商、社区类内容（我们不涉及）
  - 名称查重：需在注册时确认"买啥"未被占用，若被占用则用"买啥 MaiSha"或"MaiSha 买啥"等变体
- wxcloud 免费额度：
  - 数据库：2GB 容量 / 5 万次读写每天
  - 云函数：40 万 GBs / 月
  - 对两人使用场景远远绰绰有余

## 7. v2 / v3 路线图（仅作预留，不在 v1 实现）

### v2: 库存 + 过期提醒（+3 周）

- 新增 `inventory` collection：`{listId, name, note, purchasedAt, expiresAt, photoFileID, category, consumedAt}`
- 勾选购物清单物品时，弹出"入库存"选项 → 拍生产日期照片 → OCR 或手输入到期日期 → 写入 inventory
- 每日云函数 cron 任务：查询 `expiresAt` 在未来 3 天内的 inventory 项 → 发订阅消息推送
- 新增"库存"Tab：列表显示库存，按到期日排序，临期高亮

### v3: 菜谱 + 菜单规划（+3 周）

- 新增 `recipes` collection：`{listId, name, ingredients: [{name, quantity}], imageFileID, steps, sourceURL}`
- 新增 `meal_plans` collection：`{listId, dayOfWeek, recipeIds}`
- 菜谱录入：手工 or 从文字/链接解析
- 排菜单：选一周每天吃什么 → 汇总食材 → 对比当前 inventory → 差额一键加入购物清单

- [ ] "常买" 和 "最近添加" chips 显示正确

- [ ] 超市可自定义增删改
- [ ] 清空已购能原子执行，不会漏删或误删
- [ ] 通过微信小程序审核
- [ ] 真机（iOS + Android）全交互流畅
