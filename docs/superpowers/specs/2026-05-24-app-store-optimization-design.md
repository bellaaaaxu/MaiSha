# MaiSha App Store 优化设计文档

**日期：** 2026-05-24
**目标：** 将 MaiSha 打磨为 App Store 级别的购物清单应用
**定位：** 面向北美华人 + 中国大陆家庭用户的轻量协作购物清单
**核心理念：** 把「两个人共享一份购物清单」做到比任何 app 都简单、都好看

---

## 1. 产品定位

- **目标用户：** 华人家庭（北美 + 中国大陆），一个人规划、一个人购物的双人场景
- **核心卖点：** 零注册、分享链接即协作
- **风格：** 轻量、有温度、治愈系水彩手绘风。不做社交、不做复杂模板
- **平台：** iOS App Store（优先） + 保留 PWA（让分享对象不用装 app）
- **商业模式：** 完全免费
- **成功标准：** 几千活跃用户真心喜欢并推荐；完整走通 App Store 上架流程

---

## 2. 购物模式（Shopping Mode）

### 2.1 概述

用户进入某个超市购物时，切换到专为「在超市里看清单」设计的全屏视图。解决两个核心痛点：信息层级不清（看不清还剩什么没买）和操作不便（单手推车难操作）。

### 2.2 入口

- 每个超市卡片上增加「去购物」按钮
- 点击进入该超市的购物模式全屏页面

### 2.3 界面设计：大图标平铺

采用网格布局（3 列），每个物品以水彩图标卡片形式展示，图标是视觉主角。

**布局结构（从上到下）：**

1. **顶栏：** 返回按钮 + 超市名称 + 「结束」按钮
2. **进度条：** 横向进度条 + 「2/8」计数，用柔和动画推进（像水流）
3. **品类分区：** 按蔬菜、肉蛋、日用等分区，每区一个品类标签
4. **物品卡片：** 每张卡片包含：
   - 水彩图标（72×72，居中，圆角）
   - 物品名称（图标下方）
   - 数量角标（右上角，红色圆角标签，如「×2」）
   - 备注文字（图标下方小字，如「有机的」）
5. **「顺便看看 · 未分类」折叠区：** 未分类物品以虚线边框卡片、低透明度展示，可折叠。逻辑：在这家超市购物时顺便看看有没有能买的
6. **「已买」沉底区：** 已勾选物品卡片叠加绿色半透明遮罩 + ✓，自动沉到页面底部，透明度降低

### 2.4 交互

- **点击勾选：** 点击图标卡片即标记为已买。卡片叠加绿色遮罩 + ✓，然后柔和地「飘」到已买区（像叶子落下，不是机械移动）
- **触控区域：** 每张卡片 ≥ 80×100px，单手拇指轻松点击
- **震动反馈：** 勾选时触发轻微 haptic feedback
- **取消勾选：** 点击已买区的卡片可恢复为未买

### 2.5 结束购物

用户点击顶栏「结束」按钮或按返回时触发：

**情况 A — 有遗漏物品：**
- 弹出提示：「还有 3 件没买到」
- 两个选项：
  - 「保留下次买」— 物品留在清单中不动
  - 「全部清除」— 删除未买物品

**情况 B — 全部买完：**
- 显示水彩风格庆祝插画 + 「辛苦啦，今天的菜齐了！」
- 庆祝画面有几种随机变化（不是每次都一样）
- 「返回清单」按钮

## 3. UI 全面打磨

### 3.1 温暖文案

用对话式语气替代冷冰冰的功能文案，传达温度但不打扰：

| 场景 | 旧文案（功能型） | 新文案（温暖型） |
|------|------------------|------------------|
| 添加物品 | 添加物品 | 今天想吃什么？ |
| 空清单 | 清单为空 | 还没想好买什么，慢慢来~ |
| 购物进度 | 剩余 3 件 | 快买完啦，还差 3 样~ |
| 全部完成 | 已完成 | 辛苦啦，今天的菜齐了！ |
| 连续添加多个 | — | 哇，今天大采购！ |

### 3.2 微动画

所有动画追求「柔和、有机、像自然界的运动」：

- **勾选物品：** 图标不是机械消失，而是轻轻「飘」下去，像叶子落下
- **添加物品成功：** 水彩图标有柔和的「晕开」效果，像墨水在纸上渲染
- **进度条：** 像水流一样柔和推进，不是硬切变长
- **页面转场：** 滑入/淡出过渡，不是硬切
- **触控反馈：** 按下卡片时轻微缩放（scale 0.97），松开弹回
- **列表增删：** 物品添加、删除时有流畅的位移动画

### 3.3 小惊喜

低频出现，不打扰日常使用：

- 买完最后一件的庆祝画面随机变化（多套水彩插画）
- 连续添加多个物品时偶尔出现鼓励文案
- 不做每日打卡、不做积分系统、不做任何需要维护的游戏化机制

### 3.4 视觉打磨

- **排版系统：** 统一标题/正文/辅助文字的大小、颜色、行高
- **间距规范：** 统一使用 4px 倍数的 padding/margin
- **卡片设计：** 超市卡片层级更清晰（展开/折叠状态、物品计数角标、完成进度）
- **空状态：** 新用户空列表显示温暖引导插画 + 「添加第一件物品」提示
- **Header：** 统一图标按钮的视觉重量和间距

### 3.5 主屏 Widget（iOS）

- **有待买物品时：** 显示水彩图标缩略图 + 「还有 5 样要买~」
- **全买完时：** 温暖的水彩插画 + 「今天的菜齐啦」
- **点击行为：** 点击 Widget 直接进入购物模式
- **尺寸：** 优先支持 Medium 尺寸（2×2 格）

### 3.6 可选 App 图标

- 提供 3-4 套水彩风格的 App 图标供用户在设置中切换
- 风格可以是不同配色或季节主题

## 4. 新增功能

### 4.1 购物历史

**原则：** 自动记录，不需要用户手动操作。

- 每次「结束购物」后自动保存一条历史记录
- **历史列表页：** 按日期倒序排列，每条显示：日期 + 超市名 + 物品数量 + 水彩图标缩略图
- **一键重新添加：** 点进某次记录，查看当时买了什么，支持「再买一次」将全部或部分物品重新加回清单
- **不做统计分析：** 不做花费统计、不做图表。保持轻量

**数据模型：**
- 新增 `purchase_history` 表：`id`, `list_id`, `supermarket`, `items_snapshot`（JSON），`completed_at`
- `items_snapshot` 保存当次购买的物品快照（名称、数量、品类、图标）
- 已买和未买物品都记录（未买的标记为 `skipped`）

### 4.2 常买清单

**原则：** 从使用行为中自然产生，不需要手动维护。

- **自动统计：** 根据购物历史识别高频物品（累计出现 3 次以上的物品）
- **「常买」入口：** 在 AddSheet 图标网格最上方显示「常买」区域，展示 5-8 个最常买的物品
- **一键全部添加：** 支持「常买的都加上」快速生成基础清单
- **无模板概念：** 不做自定义模板功能。常买清单就是系统帮你记住的购物习惯

**实现方式：**
- 基于 `purchase_history` 中的 `items_snapshot` 统计物品出现频率
- 客户端计算，不需要额外后端接口
- 频率数据可缓存在 localStorage，定期从历史记录刷新

## 5. 图片资产策略

### 5.1 带品牌 Logo 的图标替换

当前部分图标使用了特定品牌商品图片（如特定牌子的牛奶、酱油等）。上架 App Store 前需全部替换为通用水彩手绘版本，避免品牌版权风险和 App Store 审核被拒。

**替换流程：**
1. 整理出所有包含品牌 logo 的图标清单
2. 为每个图标准备 Gemini 生成 prompt
3. 使用 Gemini 生成通用水彩版本替换

### 5.2 新增图片资产

以下场景需要新的水彩插画：
- 购物完成庆祝画面（3-4 套随机变化）
- 空列表引导插画
- Onboarding 引导页插画
- App 图标候选（3-4 套风格）
- Widget 背景/空状态插画

所有新增图片由用户通过 Gemini 生成，开发时提供对应的 prompt。

## 6. 不做的事情

明确排除，避免功能蔓延：

- **不做社交功能** — 没有动态、评论、点赞
- **不做复杂模板** — 没有自定义模板编辑器
- **不做花费统计** — 没有价格输入和图表分析
- **不做食谱功能** — 本次迭代不含（保留为未来 v2）
- **不做时间感知问候** — 避免过度打扰（如「早上好」「下班了」等）
- **不做积分/打卡** — 不做游戏化激励系统
- **不做国际化** — 本次只做中文版
- **不做 Android** — 本次只做 iOS + PWA

## 7. 技术注意事项

### 7.1 iOS 原生功能

购物模式、Widget、可选 App 图标、Haptic Feedback 这些功能需要 Capacitor.js 桥接层支持：

- **Haptic Feedback：** `@capacitor/haptics` 插件
- **App 图标切换：** 需要 Capacitor 插件或原生 Swift 桥接（`UIApplication.setAlternateIconName`）
- **Widget：** 需要原生 Swift WidgetKit 扩展（Capacitor 无法直接实现），数据通过 App Groups 共享
- **App Badge：** `@capacitor/local-notifications` 或 badge API

### 7.2 PWA 兼容性

购物模式和温暖交互在 PWA 中同样可用。以下功能仅限 iOS 原生版：
- Widget
- 可选 App 图标
- Haptic Feedback（PWA 中部分浏览器支持 Vibration API 作为降级方案）

### 7.3 数据层

- `purchase_history` 表需要 Supabase migration
- 常买清单统计在客户端本地计算
- 购物模式是纯前端视图切换，不需要新的后端接口

### 7.4 实施分期建议

**第一批（App Store 上架前必须）：**
- 购物模式（核心新功能）
- 温暖文案替换
- 微动画和视觉打磨
- 品牌 Logo 图标替换
- 购物历史 + 常买清单

**第二批（上架后迭代）：**
- 主屏 Widget（需要原生 WidgetKit，工作量较大）
- 可选 App 图标
- 小惊喜插画（庆祝画面多样化）
- Onboarding 引导页优化
